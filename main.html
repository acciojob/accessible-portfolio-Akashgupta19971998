ARIA Roles, States & Properties

ARIA (Assistive Rich Internet Applications), is a spec from the World Wide Web Consortium (W3C) that was created to improve accessibility of web pages and applications by providing extra information to screen readers via HTML attributes. Screen readers work with regular HTML, but adding ARIA can provide screen reader users with greater context and interactivity with the content on the page. ARIA has no effect on how elements are displayed or behave in browsers. It does not add new functionality, and is meant to act only as an extra descriptive layer for screen readers. ARIA attributes are divided into two categories: roles, and states & properties.

ARIA Roles

An ARIA role is added via a role="" attribute, and does not ever change for an element once it is set. There are four categories of ARIA roles:

  • landmarkdocument
  • document
  • widget
  • abstract
